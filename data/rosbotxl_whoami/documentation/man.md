# 萝卜头巡检机器人操作与硬件手册

本文档是对萝卜头巡检机器人的说明。内容覆盖机器人静态资料、传感器配置、导航能力、ROS 2 接口、运行限制和常见故障。

## 1. 机器人概览

萝卜头巡检机器人基于云深处绝影 X30 四足机器人二次开发设计，搭载双光云台、气体检测仪等设备，完成化工、核电等工业应急场景的巡检作业工作，可通过 RAI agent 使用工具进行交互式任务规划。

### 1.1 机身尺寸与重量

- 站立尺寸（长 × 宽 × 高）：1000 × 695 × 470 mm
- 整机重量（含电池）：56 kg

### 1.2 运动能力

- 最大运动速度：≥ 4.0 m/s
- 最大爬坡角度：≤ 45°
- 最大越障高度（台阶/障碍物）：≥ 20 cm
- 可攀爬镂空工业楼梯

### 1.3 工况要求

- 防护等级：IP67
- 工作环境温度：-20 ℃ ～ 55 ℃

### 1.4 供电与续航

- 支持自动充电
- 续航时间：约 2.5 小时
- 续航里程：≥ 10 km

## 2. 传感器与标定

萝卜头的传感器用于环境感知、定位、避障和视觉分析。

### 2.1 双光云台相机

采用海康卫视 HM-TD5528T-15/W 双光云台

#### 2.1.1 可见光模块

- 图像传感器：1/2.8" Progressive Scan CMOS
- 分辨率：2688 × 1520（4 MP）
- 光学变焦：32×
- 焦距：4.8 ～ 153 mm
- 光圈：F1.2 ～ F4.4
- 最低照度：
    - 彩色：0.05 Lux
    - 黑白：0.01 Lux
- 宽动态（WDR）：120 dB
- 光学透雾：支持
- 数字变倍：2×、4×、8×、16×

#### 2.1.2 热成像模块

- 热成像分辨率：256 × 192
- 探测器类型：氧化钒（VOx）非制冷焦平面探测器
- 像元尺寸：12 μm
- 光谱响应范围：8 ～ 14 μm
- NETD（热灵敏度）：≤ 35 mK（25 ℃，F1.0）
- 焦距：15 mm
- 光圈：F1.0
- 视场角（FOV）：11.69°（H）× 8.78°（V）
- 最小对焦距离：2.5 m
- 数字变倍：2×、4×、8×

#### 2.1.3 测温性能

- 测温范围：
    - -20 ℃ ～ 150 ℃
    - 0 ℃ ～ 550 ℃
- 测温精度：
    - ±2 ℃
    - 或读数的 ±2%（取较大值）
- 推荐目标尺寸：约 0.1 × 0.1 m
- 最远有效测温距离（0.1 × 0.1 m 目标）：约 22.5 m

### 2.2 激光雷达

- 雷达 topic：`/scan`
- 坐标系：`laser`
- 推荐扫描频率：10 Hz
- 有效测距范围：0.12 m 到 12.0 m
- 水平扫描范围：270 度
- 主要用途：Nav2 避障、3D SLAM、局部代价地图更新

雷达不能直接提供物体类别。它只能提供几何距离信息。

### 2.3 IMU 与里程计

- IMU topic：`/imu/data`
- 里程计 topic：`/odom`
- base frame：`base_link`
- odom frame：`odom`
- map frame：`map`
- 推荐 TF 查询：`map -> base_link`

IMU 和里程计用于姿态估计与短时运动估计。全局定位通常依赖 SLAM 或 AMCL，将 `map` 与 `odom` 对齐。

## 3. 运动与导航

### 3.2 Nav2 目标点导航

#### 3.2.1 目标坐标系

导航目标使用 `map` 坐标系。目标 pose 至少需要：

- `x`：地图坐标 x，单位米
- `y`：地图坐标 y，单位米
- `yaw` 或四元数 orientation：机器人朝向

## 4. ROS 2 接口速查

### 4.1 相机 Topics

- 从双光云台存储的本地图像获取

### 4.2 定位与传感器 Topics

- `/scan`：2D 激光雷达
- `/odom`：里程计
- `/imu/data`：IMU 数据

### 4.3 TF Topics

- `/tf`：动态坐标变换
- `/tf_static`：静态坐标变换

### 4.4 常用 Frames

| Frame | 含义 |
| --- | --- |
| `map` | 全局地图坐标系 |
| `odom` | 里程计连续坐标系 |
| `base_link` | 机器人底盘中心 |
| `laser` | 激光雷达坐标系 |

### 4.5 常用 Actions

| Action | 用途 |
| --- | --- |
| `navigate_to_pose` | 发送单个 Nav2 目标点 |
| `follow_waypoints` | 执行多个路径点，当前 memory demo 未直接暴露 |

## 6. 运行限制与安全规则

### 6.1 室内环境限制

- 推荐地面：平整硬质地面
- 不推荐环境：厚地毯、湿滑地面、楼梯边缘、强反光玻璃墙附近
- 推荐通道宽度：大于 0.75 m
- 推荐避障距离：至少 0.25 m

机器人靠近楼梯或湿滑区域，应提醒安全风险。

### 6.2 相机限制

双光云台相机在以下条件下效果会下降：

- 强阳光直射
- 黑色高吸收材料
- 透明玻璃或镜面反射物体
- 小于 0.25 m 或大于 6.0 m 的目标


### 6.3 导航限制

Nav2 依赖地图、定位和代价地图。如果地图未加载、TF 不连通、雷达无数据或目标点不可达，导航会失败。常见失败原因包括：

- `map -> odom` TF 缺失
- `/scan` 无数据
- 目标点在障碍物内
- 目标点超出已知地图
- 局部代价地图膨胀半径过大

## 7. 故障排查

### 7.1 无法获取相机图像

检查步骤：

1. 确认 topic `/camera/camera/color/image_raw` 是否存在。
2. 确认相机节点是否启动。
3. 确认图像频率是否大于 0。
4. 检查网络或 USB 连接。

### 7.2 无法获取机器人位置

检查步骤：

1. 确认 `/tf` 和 `/tf_static` 是否发布。
2. 查询 `map -> base_link` 是否连通。
3. 检查 SLAM 或 AMCL 是否启动。
4. 检查 `base_link` frame 名称是否与 URDF 一致。

### 7.3 导航目标不执行

检查步骤：

1. 确认 Nav2 lifecycle 节点处于 active 状态。
2. 确认 action `navigate_to_pose` 可用。
3. 确认目标 pose 使用 `map` frame。
4. 确认目标点不在障碍物或未知区域中。

## 8 视觉检测要求

机器人获取的实时图像，需要进行视觉检测，区分为以下几类:

1. 安全帽、安全服检测: 出现在视野范围里的人需要穿戴，否则返回警告
2. 跑冒滴漏检测: 设备、管道等是否出现漏液、滴液、管道破裂、冒白烟等问题，如果存在返回警告
3. 设备警示灯检测: 设备红灯/绿灯代表两台不同的备用电机，黄灯代表设备故障，如果故障返回警告
